# This file contains definitions for automatically checked and gained achievements.
# Achievements can be seen with the Achievements command, can give rewards, and 
# are tracked on players using the Tattoo internal system.
#
# The format of this file is as follows:
# The file has two sections: [PLAYER] and [ACCOUNT].  If you do not use the account
# system, only the first section will apply.
# Under each section, there is one achievement definition per line.
#
# An achievement definition is of the following form:
# [TATTOONAME]= EVENT="[EVENT]" DISPLAY="[DISPLAY]" TITLE="[TITLE]" REWARDS="[REWARDS]" ...
# 
# [TATTOONAME] is the name of the tattoo that marks the player when they reach
# this achievement.  It should be unique, and it should not end with a number,
# which would only make it temporary.
# [TITLE] is optional.  If filled in, it is a title awarded to the player who
# achieves this event.
# [REWARDS] is also an optional list of other awards, separated by >spaces<.  
# On rewards for PLAYER achievements, rewards go only to the player.  For rewards
# on an ACCOUNT achievement, rewards go to all new players or children. Each
# entry is an amount followed by a space, and an award type.  The following are
# valid reward types:
#     * amount of XP experience. Example: 10 XP
#     * amount of QP quest points. Example: 10 XP
#     * amount of currency. Example: 10 gold coins
#     * level followed by an ability id to grant a skill. The ability ID can optionally
#       have parameters in parenthesis in key=value value form.  Keys include AUTOGAIN,
#       MASK, PARMS, PREREQS.
#       Example: 17 Spell_Light
#       Example: 21 "Spell_Light(autogain=false mask=\"-class \\\"+fighter\\\"\") 
#     * level followed by a full expertise id, which is an expertise id followed by
#       the expertise level number.  This can also optionally be followed by parenthesis 
#       with key=value parameters in them.  Keys include AUTOGAIN only.
#       Example: 7 VEXING1
#       Example: 12 VEXING1(autogain=false)
# Note the spaces and use of quotes!
# [DISPLAY] is a friendly display name of this achievement.
# [EVENT] is the type of achievement event.  Each type determines the other
# parameters.  The format of each achievement event type is as follows:

# ... EVENT="KILLS" NUM=[NUMBER] ZAPPERMASK="[ZAPPERMASK]"  PLAYERMASK="[PLAYERMASK]" ...
#     [NUM] is the number of kills required to get the achievement.
#     [ZAPPERMASK] is a standard mask denoting mobs whos death gains the 
#     player a kill count towards this achievement. See archon help on 
#     Zapper Masks (AHELP ZAPPERMASKS) from the command line for 
#     syntax on zapper-masks.
#     [PLAYERMASK] is a standard mask denoting which players may gain a kill point
#     towards this achievement.  Can be used for class filtering. See archon help on 
#     Zapper Masks (AHELP ZAPPERMASKS) from the command line for 
#     syntax on zapper-masks.
# 
# ... EVENT="STATVALUE" ABOVEBELOW="[ABOVE/BELOW]" STAT="[STAT] VALUE=[VALUE] ...
#     [STAT] is the name of the player stat to check.  These are things like
#     CURRENT STRENGTH, MAXSTRENGTH, WEIGHT, CURRENT ARMOR, MANA, CURRENT MANA,
#     QUESTPOINTS, etc. As you can see, stats that have a base (unmodified) value 
#     are entered simply, where the modified score is accessed by prefixing with 
#     CURRENT.  The stat list (which can also be prefixed with CURRENT) is:
#          LEVEL, ARMOR, DAMAGE, ATTACK, WEIGHT, HEIGHT, STRENGTH, INTELLIGENCE, 
#          DEXTERITY, CONSTITUTION, CHARISMA, WISDOM, MAXSTRENGTH, MAXINTELLIGENCE,
#          MAXDEXTERITY, MAXCONSTITUTION, MAXCHARISMA, MAXWISDOM, AGE, HITS, MANA, 
#          MOVE, HUNGER, THIRST, FATIGUE, LEVEL, USES, SPEED, MONEY, QUESTPOINTS, 
#          FOLLOWERS
#     [VALUE] is the numeric value that the stat must be at or above when
#     the achievement is checked in order to gain.
#     [ABOVE/BELOW] is either ABOVE if the faction must be above the value, or 
#     BELOW if the stat must be below the value.
# 
# ... EVENT="FACTION" ID="[ID]" ABOVEBELOW="[ABOVE/BELOW]" VALUE=[VALUE] ...
#     [ID] is the ID of the faction to check 
#     [VALUE] is the numeric value that the faction must be above or below when
#     the achievement is checked in order to gain.
#     [ABOVE/BELOW] is either ABOVE if the faction must be above the value, or 
#     BELOW if the faction must be below the value.
#
# ... EVENT="EXPLORE" AREA="[AREA]" PERCENT=[PERCENT] ...
#     [AREA] is the name of the area whose player exploration percent should be
#     checked, or WORLD for the whole world.
#     [PERCENT] is the percent of the area which must be explored to gain this 
#     achievement.
#
# ... EVENT="ROOMENTER" ROOMID="[ROOMID]"
#     [ROOMID] is the fully qualified ID of the room entry to track, or a
#     comma-delimited list of room IDs that all must be entered.
#
# ... EVENT="CRAFTING" ABILITYID="[ABILITYID]" NUM=[NUM] ...
#     [ABILITYID] is the ability ID of the crafting skill to track, or *, or a
#     comma-delimited list of ability IDs
#     [NUM] is the number of times the skill must be used to create something
#     successfully before the achievement is gained.
#
# ... EVENT="MENDER" ABILITYID="[ABILITYID]" NUM=[NUM] ...
#     [ABILITYID] is the ability ID of the crafting skill to track mending with, or *,
#     or a comma-delimited list of ability IDs 
#     [NUM] is the number of times the skill must be used to mend something
#     successfully before the achievement is gained.
#
# ... EVENT="SKILLUSE" ABILITYID="[ABILITYID]" NUM=[NUM] ...
#     [ABILITYID] is the ability ID of the skill to track, or a 
#     skill type (SPELL, etc..) or a domain name (CRAFTING) 
#     or a comma-delimited list of ability IDs or types or domains.
#     [NUM] is the number of times the skill must be used
#     successfully before the achievement is gained.
#
# ... EVENT="QUESTOR" QUESTMASK="[QUESTMASK]" PLAYERMASK="[PLAYERMASK]" NUM=[NUM] ...
#     [QUESTMASK] is optional, but may contain a regular expression to filter out
#     quests according to their names.
#     [NUM] is the number of quests that match the mask which must be completed in
#     order to get this achievement.
#     [PLAYERMASK] is a standard mask denoting which players may gain a point
#     towards this achievement.  Can be used for area filtering. See archon help on 
#     Zapper Masks (AHELP ZAPPERMASKS) from the command line for 
#     syntax on zapper-masks.
#
# ... EVENT="ACHIEVER" ACHIEVEMENTLIST="[ACHIEVEMENTLIST]"
#     [ACHIEVEMENTLIST] is a space delimited list of tattoo names referring to other
#     defined achievements.  When all achievements on that list are gained, then this
#     achievement is also gained.
#
# ... EVENT="LEVELSGAINED" NUM=[NUM] PLAYERMASK="[PLAYERMASK]"
#     [NUM] is the number of levels gained to complete this achievement.
#     [PLAYERMASK] is a standard mask denoting which players may gain a level point
#     towards this achievement.  Can be used for CLASS filtering. See archon help on 
#     Zapper Masks (AHELP ZAPPERMASKS) from the command line for 
#     syntax on zapper-masks.
#
# ... EVENT="TIMEPLAYED" SECONDS=[SECONDS] PLAYERMASK="[PLAYERMASK]"
#     [SECONDS] is the number of seconds played to complete this achievement.
#     [PLAYERMASK] is a standard mask denoting which players may gain a seconds point
#     towards this achievement.  Can be used for CLASS/AREA filtering. See archon help on 
#     Zapper Masks (AHELP ZAPPERMASKS) from the command line for 
#     syntax on zapper-masks.
#
# ... EVENT="JUSTBE" PLAYERMASK="[PLAYERMASK]"
#     [PLAYERMASK] is a standard mask denoting which players automatically
#     get this achievement.  Make sure it's a narrow filter! See archon help on 
#     Zapper Masks (AHELP ZAPPERMASKS) from the command line for 
#     syntax on zapper-masks.
#
# ... EVENT="DEATHS" NUM=[NUMBER] ZAPPERMASK="[ZAPPERMASK]"  PLAYERMASK="[PLAYERMASK]" ...
#     [NUM] is the number of deaths required to get the achievement.
#     [ZAPPERMASK] is an optional standard mask denoting killer mobs whos death kills gain 
#     the player a deaths count towards this achievement. See archon help on 
#     Zapper Masks (AHELP ZAPPERMASKS) from the command line for 
#     syntax on zapper-masks.
#     [PLAYERMASK] is a standard mask denoting which players may gain a deaths point
#     towards this achievement.  Can be used for class filtering. See archon help on 
#     Zapper Masks (AHELP ZAPPERMASKS) from the command line for 
#     syntax on zapper-masks.
# 
# Some achievement events are updated every mud-hour (10 mins) automatically by 
# the system for all players on-line and in memory.
#
# Examples: 
# ORCKILLER=EVENT="KILLS" NUM=10 ZAPPERMASK="-RACE \"+ORC\" " DISPLAY="Orc Killer!"
# ACH_SAINTLY=EVENT="FACTION" DISPLAY="Have a perfectly Good alignment" TITLE="*, the Charmer" ABOVEBELOW="ABOVE" ID="ALIGNMENT.INI" VALUE=9999
# ACH_DEMONIC=EVENT="FACTION" DISPLAY="Have a perfectly Evil alignment" TITLE="*, the Demonic" ABOVEBELOW="BELOW" ID="ALIGNMENT.INI" VALUE=1
# ACH_1ROOMENTER=EVENT="ROOMENTER" DISPLAY="Enter the start room" TITLE="*, Enterer of Start" REWARDS="" PLAYERMASK="" ROOMID="START"
# ACH_EXPLORERLIMBO=EVENT="EXPLORE" AREA="Limbo" DISPLAY="Explore Limbo" TITLE="*, the Explorer of Limbo" PERCENT=50 REWARDS="1 QP 10 XP"
